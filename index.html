02

Built work

I am a carpenter. A tidy site begets tidy work.

My defaults

  • If it is unclear, it is not ready.
  • Handover notes are part of the job.

Non-negotiables

  • Safety that's practical, not performative.
  • Quality checks that prevent rework.
  • Clean site, clean finish.
03

Learning

If people are guessing, the teaching failed.

What I build in learning

  • Examples that show “good” without a lecture.
  • Rubrics and checklists that survive moderation.
  • Evidence guides that match the real work.

My bias

  • Plain language over educational jargon.
  • One page is better than five, if it is the right page.
  • Progress visibility beats vague encouragement.
04

Systems

I build tools that save time. If it doesn't, it's gone.

Some recent things built

  • An Android field app for on-site updates and progress visibility.
  • Browser tools for evidence export, scheduling, and diff checks.
  • Templates for intake, triage, and handover.

My rules

  • Simple beats clever.
  • Documentation is part of delivery.
  • Tools should be maintainable by the next person.

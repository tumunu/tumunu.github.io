Digital Forensics

An open-source digital forensics project developed by a father-son team with decades of combined experience in security, systems, and applied research.

Background

The current redition of Tumunu was started by @dupesndudsa>, an ex-IT veteran, as a way to give @synvictiona> a meaningful path into cybersecurity after finishing his degree. With no job in hand and limited entry points, the goal was simple: build tools that prove competence and fill the gaps in a resume with something real.

It's not a product pitch or training business:just a focused effort to build tools that demonstrate capability and initiative.

The What

Tumunu is an evolving collection of security and digital forensics tools built under a single brand. The goal is to create real, reliable, auditable software that supports hands-on skill development and real-world workflows. These tools are created as needed. no hype, no roadmap promises. just working code aligned with practical needs.

Current Things

  • Bit-for-bit imaging with secure read-only methodology
  • Plugin system for classification, provenance, and custom logic
  • Integrity hashing and cryptographic verification
  • CLI-first design with readable, pipeable output
  • Written in whatever language gets the job done right

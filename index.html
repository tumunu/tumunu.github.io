Background

Tumunu began as a father‑and‑son project—@dupesnduds and @synviction—to prove competence through real, open‑source tooling. No hype, no vaporware; just working code that fills practical gaps in digital forensics.

Key Features

  • Bit‑for‑bit imaging with read‑only methodology
  • Plugin framework for classification, provenance, and custom logic
  • Integrity hashing and cryptographic verification (SHA‑256, BLAKE3)
  • CLI‑first design with pipe‑friendly JSON output
  • Language‑agnostic implementation—use the best tool for the task

Get Involved

Tumunu is an active lab. Contributors, testers, and forensic educators are welcome. If Tumunu helps your workflow—or if you have a feature in mind—reach out.
